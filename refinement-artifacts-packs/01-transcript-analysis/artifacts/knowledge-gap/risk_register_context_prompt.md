# Prompt: risk_register_context.md

```text
Use #docs/transcript_context_report.md as the primary source.

Create a markdown file called:
risk_register_context.md

Purpose:
Turn unresolved architectural uncertainty into a lightweight technical risk register.

For each risk, include:
- Risk
- Why it matters
- Trigger or source of uncertainty
- Likely impact
- Mitigation or next step
- Status

Rules:
- Include only technical or context risks relevant to implementation.
- Do not inflate minor unknowns into major risks.
- Keep the register practical, not bureaucratic.

Write the file in markdown.
```
