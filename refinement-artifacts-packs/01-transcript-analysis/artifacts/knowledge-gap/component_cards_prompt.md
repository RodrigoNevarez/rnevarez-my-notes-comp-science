# Prompt: component_cards.md

```text
Use #docs/transcript_context_report.md as the primary source.

Create a markdown file called:
component_cards.md

Purpose:
Create one reusable card per important component.

For each component, include:
- Component name
- Why it matters
- What it seems to do
- Related components
- Dependencies / constraints / risks
- Relevant people / likely owners
- Open questions
- Evidence notes (short paraphrase, not quote dump)

Rules:
- Use one markdown heading per component.
- Keep each card compact and useful.
- Preserve uncertainty when the role is not fully clear.
- Do not include low-value components.

Write the file in markdown.
```
