# Prompt: ownership_map.md

```text
Use #docs/transcript_context_report.md as the primary source.

Create a markdown file called:
ownership_map.md

Purpose:
Create a practical map of who seems closest to which component or area.

For each entry, include:
- Component / area
- Likely owner or strongest explainer
- Why this person seems relevant
- Best reason to ask them
- Confidence note (high / medium / low confidence in this ownership signal)

Rules:
- Only include people when the report suggests action value.
- Do not assert ownership as fact unless clearly supported.
- Use cautious wording like "appears closest to" or "may be the best person to ask."

Write the file in markdown.
```
