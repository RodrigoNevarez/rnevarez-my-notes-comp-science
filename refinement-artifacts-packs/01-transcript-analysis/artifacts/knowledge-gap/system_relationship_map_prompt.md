# Prompt: system_relationship_map.md

```text
Use #docs/transcript_context_report.md as the primary source.

Create a markdown file called:
system_relationship_map.md

Purpose:
Build a readable map of the important systems/components and how they relate to each other.

Structure:
- Overview
- Components
- Relationship map
- Confirmed relationships
- Inferred relationships
- Unclear relationships
- Open questions

Rules:
- Focus on the most important components only.
- Separate confirmed relationships from inferred ones.
- For unclear relationships, say what is missing.
- Prefer simple directional statements such as:
  - A sends data to B
  - C depends on D
  - E appears upstream of F

Do not invent architecture that the report does not support.
Write the file in markdown.
```
