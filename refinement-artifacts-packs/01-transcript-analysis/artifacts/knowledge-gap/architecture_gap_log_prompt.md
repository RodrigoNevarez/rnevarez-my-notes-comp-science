# Prompt: architecture_gap_log.md

```text
Use #docs/transcript_context_report.md as the primary source.

Create a markdown file called:
architecture_gap_log.md

Purpose:
Turn the unresolved and partially resolved knowledge gaps from the report into a reusable tracking document.

For each gap, include:
- Gap
- Why it matters
- Type (technical or organizational)
- Severity (high, medium, low)
- Current hypothesis
- Best validation path
- Status (open / partially resolved / resolved)
- Source of truth (leave blank if unresolved)

Rules:
- Extract only meaningful gaps.
- Do not invent resolutions.
- If the report contains inferred relationships, preserve them as hypotheses, not facts.
- Keep entries concise and scannable.

Output:
Write the file in markdown.
```
