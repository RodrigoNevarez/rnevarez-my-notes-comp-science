# Prompt: integration_points.md

```text
Use #docs/transcript_context_report.md as the primary source.

Create a markdown file called:
integration_points.md

Purpose:
Extract the important integration boundaries and handoff points across systems/components.

For each integration point, include:
- Systems/components involved
- Nature of the interaction
- What seems to flow across the boundary
- Whether the relationship is confirmed or inferred
- Risks / constraints / unknowns
- What should be verified next

Rules:
- Focus on APIs, events, queues, jobs, handoffs, and dependency boundaries.
- Do not list trivial relationships.
- Preserve uncertainty clearly.

Write the file in markdown.
```
