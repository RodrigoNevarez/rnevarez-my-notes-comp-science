# Report Section Guidelines

## 1. Important Components
Purpose: identify only the components most likely to affect the task.

For each component:
- Lead with **Why it matters**
- Explain its role and relationships
- Include meaningful dependencies, constraints, risks, and caveats
- Mention relevant people or likely owners only if useful
- Add inference only when needed
- Add a short Evidence subsection with 2–4 short excerpts

## 2. System Picture
Purpose: synthesize how the important parts fit together.

Guidelines:
- Keep it short
- Focus on the likely shape of the system
- Explain the most important relationships
- Include careful inference when relationships are implied
- Do not restate all component sections verbatim

## 3. Important but Insufficiently Explained
Purpose: surface the missing context that blocks understanding.

For each item:
- explain why it seems important
- explain what remains unclear
- explain why that gap matters to the task

## 4. Possible Next Things to Investigate
Purpose: convert ambiguity into concrete next steps.

Good entries include:
- code or docs worth checking
- people worth asking
- relationships that need confirmation
- important acronyms or services to decode
- unresolved decisions worth tracing

## General style
- Write like a strong technical analyst
- Be selective
- Avoid clutter
- Prefer clarity over completeness
- Do not fabricate certainty
