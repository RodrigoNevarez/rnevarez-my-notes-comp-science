# One-Page Cheatsheet

## Goal
Turn one meeting transcript into a task-relevant technical context report.

## Inputs
- `TASK_CONTEXT`
- `OPTIONAL_FOCUS_AREAS`
- `IGNORE_AREAS`
- `MAX_COMPONENTS`
- `TRANSCRIPT`

## Output sections
1. Important Components
2. System Picture
3. Important but Insufficiently Explained
4. Possible Next Things to Investigate

## What to optimize for
- task relevance
- architectural understanding
- relationships between components
- selective evidence
- useful ambiguity
- actionable follow-up

## What to avoid
- generic meeting summaries
- exhaustive component lists
- long quote dumps
- fake certainty
- noisy ownership mentions

## Best defaults
- one transcript at a time
- analyst notes
- 5 max components
- 2–4 short evidence excerpts per component
- inference only when clearly labeled
